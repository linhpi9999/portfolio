AI Applications: Workflow Ecosystem

A production‑ready ecosystem of AI workflows that automate operations, support customer care, and accelerate lookup, classification, and data normalization — grouped into three clusters: Chatbots, Information Retrieval, and AI Ops Platform.

Overview Chatbots Retrieval Ops Platform Key Metrics →

1) Overview

We built AI workflows to reduce manual handling, standardize decision logic, and provide consistent answers across channels. The ecosystem is organized into three product clusters: Chatbots (customer service & internal support), Batch Lookup (template‑based research), and AI Operations Platform (agents + tooling for workflow orchestration and data normalization).

What this ecosystem delivers
  • Load reduction for Customer Service via self‑serve responses and structured flows
  • Faster operations with automation for repetitive, policy‑driven tasks
  • Cleaner data through normalization, tagging, and taxonomy classification
  • Traceable decisioning using workflow nodes, intents, and guarded fallbacks

3) Chatbots (Customer Service & Internal)

The chatbot cluster focuses on guided issue handling, order tracking, and fast policy answers, with guardrails that hand over to humans when sentiment is negative or intent/order context is missing.

GC Bot

Supports customer service agents in shop support: order issues, delivery follow-ups, basic policies, and incident intake.

Chatflows 2 Workflows 7 (L2) + 34 (L3) Intents 9 (L1) + 50 (L2) Nodes ~300
  • Default handover rules: sentiment < 3/10, unknown intent, or missing order code
  • Popular requests (delivery follow-up, COD, re-delivery): > 40% of traffic
  • Strong answer rates: common requests often > 40%; info queries often > 70%
  • Disable rate: ~60% (handover to shop/human when needed)

GAM Bot

Shop-side chatbot for end-user conversations (sales, order changes, after-sales, product/shop information, logistics). Development is currently paused.

Chatflows 2 Workflows 6 (L2) + 28 (L3) Intents 12 (L1) + 34 (L2)
  • Default handover: sentiment < 3/10, or unknown intent
  • Designed for conversion and shop support automation
  • Status: paused / pending restart

Internal “Know‑It‑All”

Internal assistant for employees (e.g., HR) that automatically detects intent and routes requests to the right lookup/workflow.

Deployed intents 42 Primary goal Auto‑routing
  • Intent-first routing to reduce time-to-answer for internal queries
  • Designed to plug into knowledge bases and operational tools
Reliability and safety rails
  • Intent detection: structured intent groups (L1/L2) to reduce ambiguity
  • Fallback logic: unknown intent routed to human/shop handling
  • Sentiment-based guardrails: negative sentiment triggers escalation
  • Operational resilience: retries + monitoring; ~1% workflow errors, mainly API retries
Opportunity: reduce “Unknown/Other” intents (~15–20%) by expanding intent coverage and adding lightweight clarification questions.

4) Information Retrieval (Batch Lookup)

A template-based “ChatGPT/Perplexity style” workflow that processes a list of entities and fields, returning a consolidated output file. Designed for scalable research tasks (e.g., company profiling, locality attributes) with consistent schema and export.

Typical input → output
  • Input: list of entities (shops / localities) + list of fields to retrieve
  • Process: query → validate → enrich → normalize
  • Output: one consolidated file (table) ready for analysis

5) AI Ops Platform & Operational Workflows

This cluster provides reusable building blocks that connect insights, data sources, and execution into operational workflows — including agent-based routing, vision data normalization, tagging, and taxonomy classification.

Solution Assistant (Tool Flow) In progress

Automatically analyzes requests, recommends the right workflow, and pushes execution to an ops workflow platform. Implemented as a 4-agent pipeline.

Agents 4 Goal auto-routing Output workflow plan
  • Study Requirement → parse intent, constraints, scope
  • DA & AI → connect data + produce insights
  • Workflow Generator → build/compose workflow steps
  • QA/QC → evaluate and suggest improvements

Vision Config Normalization In progress

Normalizes OCR/raw output using a configuration file: correct types, formats, structures, and merges back into the vision response.

Input config + vision_response Output standardized JSON
  • Transform config/dataset → LLM normalize → parse/decompose → merge
  • Assign back to vision_response with consistent schema
  • Designed for downstream automation and validation

Insight → Solution Mapping In progress

Maps structured insights to recommended solutions using a knowledge base: object/attributes → inferred solutions.

Recall ~30% Precision >80% Method object + attribute pairs
  • Normalize object/attribute → (object, attribute_value)
  • Infer matching solution candidates from KB
  • Prioritize precision to reduce wrong recommendations

Shop-name Tag Extraction In progress

Detects tags and tag values from shop_name (one shop → multiple tags; one tag → multiple values; comma-separated).

Fields 7/7 Accuracy >90% Note runtime improving
  • Improved “production-based model” accuracy: 22.85% → 94.1%
  • Improved “HQ/Branch” accuracy: 66.7% → 100%
  • Current limitation: longer runtime

6) Next steps

Near-term priorities
  • Reduce “Unknown/Other” intents by extending coverage and adding clarification flows
  • Improve Insight → Solution recall while keeping precision high
  • Optimize runtime for shop-name tag extraction and add monitoring
  • Standardize evaluation and release criteria across all workflows
What makes it scalable
  • Reusable workflow templates and shared guardrails
  • Versioned intents / configs with traceable changes
  • Metrics-first monitoring: volume, fallback rate, error rate, precision/recall
  • Design for safe handover to humans in sensitive cases
