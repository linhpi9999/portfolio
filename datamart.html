Datamart Case Study — Lakehouse Architecture with Apache Iceberg

Mục tiêu của dự án là xây dựng một nền tảng dữ liệu thống nhất, có khả năng mở rộng và tối ưu cho analytics/BI. Giải pháp sử dụng Lakehouse + Apache Iceberg nhằm đảm bảo truy vấn ổn định, quản trị dữ liệu tốt và hỗ trợ cả batch lẫn streaming.

1) Project overview

Dự án Datamart giúp chuẩn hoá luồng dữ liệu từ source → landing → bronze/silver → gold (datamarts), phục vụ dashboard/analytics với tiêu chí: đúng, nhanh, ổn định.

Problems we solved

  • Logic marts rời rạc, khó kiểm soát chất lượng
  • Query chậm do partition/format chưa tối ưu
  • Thiếu chuẩn hoá schema & layer hoá dữ liệu

What we built

  • Gold datamarts theo business domain
  • Chuẩn hoá mô hình (facts/dims), KPI definitions
  • Monitoring/validation cho chất lượng dữ liệu

2) Architecture overview

Kiến trúc gồm Lake storage + file format + table format (Iceberg) + compute engines (Spark/Trino) + catalog. Thiết kế này giúp quản trị phiên bản bảng, schema evolution và tối ưu truy vấn.

3) Dataflow: source → gold layer

Layering strategy

  • Landing: raw ingestion, giữ nguyên cấu trúc
  • Bronze: chuẩn hoá schema tối thiểu
  • Silver: clean, dedupe, consolidate
  • Gold: datamarts theo use-case/KPI

4) My role & deliverables

Datamart design

  • Translate business needs → logical model (facts/dims)
  • Định nghĩa KPI, grain, business rules, mapping
  • Thiết kế partitioning/aggregation để tối ưu query

Quality & performance

  • Review/optimize marts cũ: logic, partition, compute cost
  • Data checks: completeness, uniqueness, anomaly signals
  • Documentation: metric definitions & data contracts

5) Impact

Kết quả chính tập trung vào ổn định vận hành và nâng chất lượng phục vụ analytics. (Bạn có thể thay bằng số liệu thật nếu muốn “wow” hơn.)

Business outcomes

  • Giảm độ trễ refresh & tăng độ tin cậy số liệu
  • KPI nhất quán giữa nhiều dashboard/report
  • Enable use-cases: BI, realtime, ML pipelines

Tech stack

  • Storage: S3/Azure/GCS
  • Formats: Parquet/Avro
  • Table format: Apache Iceberg
  • Engines: Spark/Trino
  • Catalog: Hive/Glue/Nessie
