Datamart Project – Lakehouse Architecture with Apache Iceberg

1. Project Overview

The Datamart project was built to create a unified, scalable, and high-performance data platform. By adopting a Lakehouse architecture combined with Apache Iceberg, the system ensures:

  • Fast, stable, and consistent data queries
  • Flexible support for both batch and streaming processing
  • Scalability for data products and AI/ML pipelines

I contributed as an Analytics Engineer, responsible for logic design, data modeling, and building/maintaining Datamarts for analysis.

2. Architecture Overview

The system follows a multi-layered architecture including:

  • Lake Storage: S3, Azure, GCS
  • File formats: Parquet, Avro, Lance
  • Table format: Apache Iceberg
  • Compute engines: Spark, Trino
  • Catalogs: Hive, AWS Glue, Nessie

3. Dataflow – From Source to Gold Layer

Data is ingested and transformed through multiple layers:

  • Landing Zone: Raw data in original structure
  • Bronze Layer: Basic schema standardization
  • Silver Layer: Data cleaning and consolidation
  • Gold Layer: Final Datamarts optimized for analytics

4. My Role in the Project

  • Operate & optimize legacy marts: reviewing logic, optimizing partitions & performance
  • Design & build new marts: translating business needs into logical models
  • Ensure data quality: documentation, transformation testing, data monitoring
  • Collaborate with Data Engineering: schema standardization, pipeline reviews

5. Key Achievements

  • Significantly reduced data refresh latency
  • Ensured consistent and reliable Datamart quality
  • Enabled downstream systems including BI, realtime dashboards, ML pipelines
